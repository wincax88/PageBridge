## Design Context

### Users
PageBridge / 阅迹 serves students, researchers, and focused knowledge workers who read PDFs, add highlights and notes, and continue across devices. Their primary jobs are uploading documents, resuming recent reading, annotating important passages, and trusting that progress and notes sync reliably.

### Brand Personality
Professional, calm, and reading-friendly. The interface should feel like a precise PDF productivity tool: low distraction, clear sync state, high information density, and strong trust cues.

### Aesthetic Direction
Clean light SaaS workspace based on the Figma design: `#F8FAFC` app background, white panels, `#2563EB` primary actions, thin `#E5E7EB` borders, compact Chinese UI copy, Inter / Noto Sans SC typography, and rounded but restrained controls.

### Design Principles
- Keep reading central: tools should support the PDF canvas without competing for attention.
- Make sync visible: every file and annotation workflow should show whether changes are synced, queued, or failed.
- Use dense but legible layouts: desktop should expose navigation, document metadata, and annotation context at a glance.
- Prefer familiar tool patterns: sidebar navigation, top toolbars, cards/lists, and inspector panels should feel predictable.
- Preserve calm visual hierarchy: blue only for primary actions and active state, with neutral surfaces for sustained reading.
