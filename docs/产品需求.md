# 项目需求：PageBridge 跨平台 PDF 阅读与标注同步工具

## 1. 产品定位

PageBridge 是一款面向个人深度阅读场景的 PDF 阅读、标注与多端同步工具。首版重点解决用户在不同设备之间继续阅读、查看标注、同步批注的连续性问题，而不是成为完整的团队协作平台或文档知识库。

### 1.1 核心价值

| 价值 | 说明 |
| --- | --- |
| 跨端连续阅读 | 用户在一台设备上阅读和标注后，可在另一台设备继续阅读 |
| 标注独立保存 | 不直接破坏原始 PDF，标注以结构化数据保存和同步 |
| 本地优先体验 | 阅读和标注优先即时落本地，网络恢复后再同步 |
| 可控云同步 | 文件、阅读进度、标注、笔记可自动同步，并展示明确同步状态 |

### 1.2 首版目标用户

| 用户类型 | 典型场景 | 首版优先级 |
| --- | --- | --- |
| 学生 | 阅读教材、论文、课件，做高亮和笔记 | P0 |
| 研究人员 | 阅读论文、整理标注、跨设备继续阅读 | P0 |
| 职场用户 | 审阅报告、合同、方案文档 | P1 |
| 团队用户 | 多人批注文档、评论回复、协作审阅 | P3 |

### 1.3 非目标

MVP 阶段明确不做以下能力：

1. 多人实时协作标注。
2. OCR 识别、扫描件全文搜索和扫描件文本高亮。
3. AI 总结、问答、知识库和文献管理。
4. 复杂图形编辑、签章、表单填写和 PDF 内容编辑。
5. 企业级权限、组织空间、审计后台和 SSO。

## 3. MVP 范围

### 3.1 MVP 必做能力

| 模块 | 功能 |
| --- | --- |
| 账号 | 邮箱注册、邮箱登录、退出登录、登录态刷新 |
| 文件管理 | 上传 PDF、文件列表、重命名、删除、按文件名搜索 |
| PDF 阅读 | 打开 PDF、翻页、连续滚动、缩放、适应宽度、页码跳转、文本搜索、目录导航 |
| 标注 | 文本高亮、文本批注、颜色选择、删除标注、标注列表 |
| 阅读进度 | 保存最近阅读页码、滚动位置、缩放模式 |
| 同步 | 文件同步、阅读进度同步、标注同步、失败提示、手动重试 |
| 离线 | 已打开文件可继续查看和新增标注，联网后同步变更 |

### 3.2 MVP 暂不做能力

| 功能 | 原因 |
| --- | --- |
| 自由画笔、图形标注 | 跨设备坐标、触控体验和性能复杂度较高 |
| 扫描 PDF OCR | 成本高，且不是首版核心同步验证能力 |
| 分享链接和协作批注 | 涉及权限、审计、通知和冲突复杂度 |
| 回收站和版本历史 | 可后置，不影响核心阅读和同步闭环 |
| 移动端深度标注 | 文本选择和触控标注体验需要单独设计 |

## 4. 核心用户流程

### 4.1 首次使用流程

1. 用户打开 Web 应用。
2. 用户注册或登录账号。
3. 用户上传一个 PDF 文件。
4. 系统进入阅读器并展示第一页。
5. 系统在文件列表中展示该 PDF。

### 4.2 阅读与标注流程

1. 用户打开已上传 PDF。
2. 用户搜索关键词或通过目录定位页面。
3. 用户选中文本并添加高亮。
4. 用户为高亮添加文本批注。
5. 客户端立即本地保存标注并显示“同步中”。
6. 服务端保存成功后显示“已同步”。

### 4.3 多端继续阅读流程

1. 用户在设备 A 阅读到第 20 页并添加标注。
2. 设备 A 将阅读进度和标注同步到云端。
3. 用户在设备 B 登录同一账号。
4. 设备 B 打开同一文件后恢复到第 20 页。
5. 设备 B 展示设备 A 已添加的标注。

### 4.4 离线编辑流程

1. 用户已打开过某 PDF，客户端已缓存文件和标注数据。
2. 网络断开后，用户继续阅读并新增高亮。
3. 客户端将变更写入本地离线队列，标记为“待同步”。
4. 网络恢复后，客户端按顺序上传离线变更。
5. 如果服务端接受变更，状态更新为“已同步”。
6. 如果发生冲突，状态更新为“同步冲突”，并按冲突策略处理。

```json
    [120.0, 300.0, 360.0, 300.0, 120.0, 318.0, 360.0, 318.0]

    "x": 120.0,
    "y": 300.0,

    "height": 18.0

  "page_height": 792.0,
  "page_rotation": 0,
  "version": 3,
  "deleted_at": null,
  "created_at": "2026-05-10T10:00:00Z",
  "updated_at": "2026-05-10T10:05:00Z"
}
```

客户端和服务端通过变更日志实现增量同步。

```json
{
  "change_id": "chg_001",
  "user_id": "user_001",
  "file_id": "pdf_001",
  "entity_type": "annotation",
  "entity_id": "ann_001",
  "operation": "update",
  "base_version": 2,
  "next_version": 3,
  "client_request_id": "req_001",
  "created_at": "2026-05-10T10:05:00Z"
}
```

## 7. 标注坐标规范

1. 所有标注坐标使用 PDF 页面原始坐标系，不使用浏览器像素坐标作为持久化坐标。
2. 文本高亮必须保存 `quad_points`，支持跨行高亮。
3. `rect` 仅作为快速命中检测和列表预览的辅助字段，不作为唯一定位依据。
4. 必须保存 `page_width`、`page_height` 和 `page_rotation`，用于不同渲染尺寸下还原位置。
5. 缩放、滚动和设备 DPI 只影响渲染，不影响服务端保存的坐标。
6. 页面旋转只影响阅读视图时，不修改原始坐标；如果未来支持永久旋转，需要生成独立变更。
7. 扫描型 PDF 无文本层时，不支持文本高亮，只支持页面位置型批注。

## 8. 同步设计

### 8.1 同步原则

| 原则 | 说明 |
| --- | --- |
| 本地优先 | 用户操作必须立即在本地生效 |
| 增量同步 | 客户端只上传变更，不重复上传全量标注 |
| 幂等提交 | 每次变更包含 `client_request_id`，服务端重复收到时只处理一次 |
| 软删除 | 删除标注和文件优先写入 `deleted_at`，避免离线同步丢失上下文 |
| 可恢复失败 | 网络失败不丢失本地变更，用户可手动重试 |

### 8.2 同步流程

1. 用户执行上传、标注、删除、重命名等操作。
2. 客户端先写入本地数据库或缓存，并生成本地变更记录。
3. 客户端将变更加入同步队列，状态为“同步中”或“待同步”。
4. 服务端校验权限、版本号和配额。
5. 服务端接受变更后返回新的 `version` 和服务端时间。
6. 客户端更新本地实体版本和同步状态。
7. 其他设备通过增量拉取接口获取更新。

### 8.3 冲突处理

| 场景 | 策略 |
| --- | --- |
| 两端新增不同标注 | 直接合并 |
| 两端修改同一标注不同字段 | 尽量字段级合并，例如颜色和备注可合并 |
| 两端修改同一标注同一字段 | 默认以后提交者覆盖，并记录冲突日志 |
| 一端删除、一端修改 | 删除优先；客户端提示“该标注已在其他设备删除” |
| 离线变更基于旧版本 | 服务端返回冲突，客户端拉取最新版本后重放可合并变更 |

### 8.4 同步接口建议

| 接口 | 说明 |
| --- | --- |
| `POST /files` | 上传 PDF 元数据并获取上传地址 |
| `GET /files` | 获取文件列表 |
| `GET /files/{file_id}/changes?since=` | 拉取某文件自指定游标后的变更 |
| `POST /sync/changes` | 提交客户端变更 |
| `GET /sync/state` | 获取账号级同步游标 |

## 9. 非功能需求

### 9.1 性能要求

| 项目 | MVP 指标 |
| --- | --- |
| 50MB PDF 首屏加载 | P75 小于 3 秒，P95 小于 6 秒 |
| 200MB PDF 打开 | 可打开并逐页加载，不出现浏览器崩溃 |
| 翻页响应 | P75 小于 200ms |
| 缩放响应 | P75 小于 300ms |
| 搜索响应 | 100 页文本型 PDF 关键词搜索 P75 小于 2 秒 |
| 标注创建反馈 | 本地 UI 反馈小于 100ms |
| 同步延迟 | 网络正常时标注变更 P75 小于 2 秒同步完成 |

### 9.2 可用性要求

| 项目 | 要求 |
| --- | --- |
| 状态明确 | 上传、同步、离线、失败、冲突均有明确提示 |
| 错误可恢复 | 上传失败、同步失败可重试 |
| 操作可撤销 | 删除标注后短时间内支持撤销，或至少提供确认 |
| 快捷键 | 桌面 Web 支持搜索、翻页、缩放常用快捷键 |
| 空状态 | 无文件、无搜索结果、无标注时有明确引导 |

### 9.3 安全与隐私

| 项目 | 要求 |
| --- | --- |
| 传输加密 | 所有 API、上传、下载必须使用 HTTPS |
| 存储加密 | 文件存储使用对象存储服务端加密或等价能力 |
| 权限隔离 | 所有文件、标注、进度查询必须校验 `user_id` |
| Token 安全 | access token 短有效期，refresh token 可撤销 |
| 删除策略 | 用户删除文件后进入 7 天软删除，之后物理删除文件和标注 |
| 隐私保护 | 未经用户授权，不分析、不训练、不分享用户 PDF 内容 |
| 备份 | 服务端数据库定期备份，文件存储开启冗余策略 |

MVP 不承诺端到端加密。如果未来支持零知识存储，需要重新设计搜索、预览和同步模型。

### 9.4 兼容性要求

| 项目 | 要求 |
| --- | --- |
| PDF 类型 | 文本型 PDF、扫描型 PDF、带目录 PDF |
| 文本层 | 文本搜索和文本高亮仅对带文本层 PDF 承诺 |
| 字体和编码 | 尽量支持常见嵌入字体、中文 PDF 和英文论文 |
| 大文件 | 支持最大 200MB 单文件 |

### 10.2 标注验收

| 验收项 | 标准 |
| --- | --- |
| 添加高亮 | 用户可选中文本并添加高亮 |

### 10.3 同步验收

| 验收项 | 标准 |
| --- | --- |
| 文件同步 | 设备 A 上传的 PDF，设备 B 登录后可查看 |

| 离线同步 | 离线期间新增标注，联网后自动同步 |
| 同步失败提示 | 网络异常时显示失败状态，并支持重试 |
| 冲突处理 | 同一标注并发修改时不会丢失本地变更且有明确提示 |

### 10.4 安全验收

| 验收项 | 标准 |
| --- | --- |
| 权限隔离 | 用户不能访问其他用户文件、标注和进度 |
| 未登录访问 | 未登录用户不能访问私有文件 API |
| 删除验证 | 删除文件后普通列表不再返回该文件 |
| Token 过期 | access token 过期后可刷新或要求重新登录 |

## 11. 主要风险点

| 风险 | 说明 | 应对方案 |
| --- | --- | --- |
| PDF 渲染性能 | 大文件、复杂 PDF 渲染慢 | 分页加载、懒渲染、页面缓存、使用成熟 PDF 引擎 |
| 标注定位偏移 | 缩放、旋转、不同渲染尺寸可能导致偏移 | 使用 PDF 页面坐标和 `quad_points` |
| 多端冲突 | 多设备同时编辑同一标注可能冲突 | 引入版本号、变更日志、软删除和冲突提示 |
| 离线可靠性 | 离线队列丢失会导致用户标注丢失 | 本地持久化队列和幂等重试 |
| 扫描件能力预期 | 扫描 PDF 无文本层，无法文本搜索和高亮 | 在产品文案中明确能力边界 |
| 云存储成本 | PDF 文件体积大，存储和流量成本高 | 设置配额、限制单文件大小、后续引入付费计划 |
| 数据安全 | 用户文档可能包含敏感信息 | HTTPS、存储加密、权限隔离、删除策略和备份 |

## 12. 推荐技术方案方向

### 12.1 前端

| 场景 | 技术方向 |
| --- | --- |
| Web 端 | React / Vue + PDF.js |
| PDF 渲染 | PDF.js 优先，必要时评估 MuPDF 或商业 SDK |
| 标注层 | SVG Overlay 优先，Canvas 用于高性能批量渲染场景 |
| 本地缓存 | IndexedDB 保存离线队列、进度和标注缓存 |
| 桌面端 | MVP 后使用 Tauri 或 Electron 封装 Web 能力 |

### 12.2 后端

| 模块 | 技术方向 |
| --- | --- |
| API 服务 | Node.js / Go / Java / Python |
| 数据库 | PostgreSQL / MySQL |
| 文件存储 | S3、OSS、COS、MinIO |
| 同步服务 | REST 增量同步接口，后续可增加 WebSocket 推送 |
| 鉴权 | JWT + refresh token |
| 缓存 | Redis 用于会话、限流和热点数据 |

## 14. 一句话产品定义

PageBridge 是一款本地优先、云端同步的 PDF 阅读与标注工具，让用户可以在不同设备上连续阅读、保存批注，并可靠同步阅读进度和标注内容。
