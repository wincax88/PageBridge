下面是一版 **「阅迹」移动端 H5 界面设计方案**，按手机浏览器场景设计，适合作为移动端原型、UI 设计稿和前端开发说明。

设计基准建议：

```txt
设计稿尺寸：390 × 844
兼容范围：360px - 430px 宽度
页面类型：移动端 H5 / PWA
主要场景：手机浏览器阅读 PDF、查看标注、轻量编辑、同步阅读进度
```

# 一、移动端 H5 设计原则

PC Web 可以做成“左侧导航 + 中间 PDF + 右侧标注面板”的工作台，但移动端空间有限，不能照搬 PC 布局。

移动端应该采用：

```txt
单列主界面
顶部轻导航
底部主操作区
抽屉式面板
浮动标注工具
阅读优先，工具后置
```

核心原则：

| 原则   | 说明                   |
| ---- | -------------------- |
| 阅读优先 | PDF 页面尽量占满屏幕，工具栏自动收起 |
| 操作分层 | 常用操作放底部，次要操作放抽屉或更多菜单 |
| 减少误触 | 标注、选择、拖动、缩放需要明确区分    |
| 同步可见 | 标注和阅读进度同步状态需要清晰提示    |
| 轻量标注 | 移动端优先做高亮、文字批注，复杂画笔后置 |
| 支持离线 | 允许用户离线阅读和添加标注，联网后同步  |

# 二、移动端页面结构

移动端 H5 建议包含以下页面：

| 页面       | 路由                | 优先级 | 说明             |
| -------- | ----------------- | --- | -------------- |
| 启动 / 登录页 | `/login`          | P0  | 登录、注册、找回密码     |
| 文件库首页    | `/library`        | P0  | 查看、搜索、上传 PDF   |
| 最近阅读页    | `/recent`         | P0  | 快速继续上次阅读       |
| PDF 阅读页  | `/reader/:fileId` | P0  | 阅读、缩放、标注、搜索、同步 |
| 标注列表页    | `/annotations`    | P1  | 汇总所有 PDF 标注    |
| 设置页      | `/settings`       | P1  | 账号、同步、阅读偏好     |
| 文件详情页    | `/file/:fileId`   | P1  | 文件信息、重命名、删除、分享 |

MVP 阶段建议优先完成：

```txt
/login
/library
/reader/:fileId
/settings
```

# 三、整体导航设计

移动端非阅读页面建议使用底部 Tab。

```txt
┌──────────────────────┐
│ 页面内容              │
│                      │
│                      │
├──────────────────────┤
│ 文档  最近  标注  我的 │
└──────────────────────┘
```

底部 Tab：

| Tab | 说明         |
| --- | ---------- |
| 文档  | 文件库首页      |
| 最近  | 最近阅读       |
| 标注  | 所有标注       |
| 我的  | 设置、账号、存储空间 |

但 **PDF 阅读页不使用底部 Tab**，否则会占用阅读空间。阅读页使用独立的沉浸式布局。

# 七、移动端标注设计

移动端标注是难点，因为手机上的文本选择、缩放、滚动很容易互相干扰。

建议采用两种模式：

```txt
阅读模式
标注模式
```

## 7.1 阅读模式

默认状态。

用户可以：

```txt
滚动
缩放
查看标注
点击批注
搜索
打开目录
```

不直接进入复杂标注，避免误触。

---

## 7.2 标注模式

点击底部“标注”后，打开标注工具抽屉：

```txt
┌────────────────────────┐
│ 选择标注工具             │
│                        │
│ 🟨 高亮文本              │
│ 📝 添加批注              │
│ ✏️ 自由画笔              │
│ 🔖 添加书签              │
│                        │
│ 退出标注模式             │
└────────────────────────┘
```

MVP 阶段推荐优先支持：

```txt
高亮文本
文字批注
书签
```

暂缓：

```txt
自由画笔
图形标注
下划线
删除线
```

---

## 7.3 选中文本后的浮动工具栏

用户长按选择文本后，显示浮动工具栏。

```txt
┌────────────────────┐
│ 高亮  批注  复制    │
└────────────────────┘
```

如果空间不够，可以做成底部操作条：

```txt
┌────────────────────────┐
│ 已选择文本               │
│ [高亮] [批注] [复制]      │
└────────────────────────┘
```

我更推荐移动端使用**底部操作条**，比浮动小工具栏更稳定。

### 选中文本后的流程

```txt
长按选择文本
  ↓
系统显示选区
  ↓
底部出现操作条
  ↓
点击高亮 / 批注
  ↓
保存标注
  ↓
显示同步状态
```

# 十二、同步状态设计

# 十九、移动端 H5 特别注意点

## 1. 使用 `100dvh`

移动端浏览器地址栏会动态收起，如果直接使用 `100vh`，可能导致底部工具栏被遮挡。

建议使用：

```css
height: 100dvh;
```

并处理安全区域：

```css
padding-bottom: env(safe-area-inset-bottom);
```

---

## 2. 阅读页底部工具栏避开安全区

```txt
底部工具栏高度：56px + safe-area
```

iPhone 底部有 Home Indicator，按钮不能贴到底。

---

## 3. PDF 页面不要一次性渲染太多

移动端性能有限，建议：

```txt
只渲染当前页附近的页面
当前页前后各 1-2 页保持活跃
远距离页面只保留占位高度
```

---

## 4. 标注坐标必须归一化

不要存屏幕像素坐标。

应该存：

```txt
x = rect.x / pageWidth
y = rect.y / pageHeight
width = rect.width / pageWidth
height = rect.height / pageHeight
```

这样不同屏幕、不同缩放比例都能正确还原标注位置。

---

## 5. 区分滚动和标注

移动端最容易出现的问题是：

```txt
用户想滚动，但触发标注
用户想选文本，但页面滑动
用户想缩放，但标注层拦截触摸
```

所以建议：

```txt
默认阅读模式下：优先滚动和缩放
进入标注模式后：才启用更多标注交互
```

# 最终建议

移动端 H5 不要设计成缩小版 PC 阅读器，而应该设计成：

> **一个以阅读为主、以底部抽屉承载工具和标注管理的轻量 PDF 阅读界面。**

推荐移动端主结构是：

```txt
文件库：顶部搜索 + 文件列表 + 底部 Tab
阅读器：顶部文件栏 + 中央 PDF + 底部工具栏 + 抽屉面板
标注：长按选区 + 底部操作条 + 批注抽屉
同步：顶部状态图标 + Toast 提示
```

这样既能保留 PDF 阅读和标注能力，又不会让移动端界面过于拥挤。
