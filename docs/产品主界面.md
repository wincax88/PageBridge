以下是 **「阅迹」PC Web 端页面设计方案**，按 1440×900 桌面端尺寸设计，适合作为产品原型、UI 设计稿和前端开发说明的基础。

# 一、整体页面架构

产品核心是：

> 文件管理 → 打开 PDF → 阅读 → 标注 → 自动同步 → 多端继续阅读

PC Web 端建议包含以下页面：

| 页面       | 路由                | 优先级 | 说明                    |
| -------- | ----------------- | --- | --------------------- |
| 登录 / 注册页 | `/login`          | P0  | 用户登录、注册、找回密码          |
| 文件库首页    | `/library`        | P0  | 管理 PDF 文件、上传文件、查看最近阅读 |
| PDF 阅读器页 | `/reader/:fileId` | P0  | PDF 浏览、标注、搜索、同步       |
| 标注管理页    | `/annotations`    | P1  | 汇总查看所有文档标注            |
| 回收站页     | `/trash`          | P1  | 恢复或彻底删除文件             |
| 设置页      | `/settings`       | P1  | 账号、存储、同步、快捷键设置        |

MVP 阶段重点做：

```txt
/login
/library
/reader/:fileId
/settings
```

# 五、页面 3：PDF 阅读器页

这是最重要的页面。

## 页面目标

用户可以高效阅读 PDF，进行高亮、批注、笔记，并实时同步。

## 总体布局

推荐采用：

```txt
顶部工具栏 + 左侧文档导航 + 中央 PDF 阅读区 + 右侧标注面板
```

完整结构：

```txt
┌────────────────────────────────────────────────────────────────────┐
│ 顶部工具栏：返回 / 文件名 / 页码 / 缩放 / 搜索 / 标注工具 / 同步状态    │
├───────────────┬──────────────────────────────────┬─────────────────┤
│ 左侧导航栏      │ 中央 PDF 阅读区                    │ 右侧标注面板       │
│               │                                  │                 │
│ 缩略图          │        PDF 页面                    │ 标注列表          │
│ 目录            │                                  │ 笔记              │
│ 书签            │                                  │ 搜索结果          │
│               │                                  │                 │
└───────────────┴──────────────────────────────────┴─────────────────┘
```

推荐尺寸：

| 区域     | 宽度 / 高度   |
| ------ | --------- |
| 顶部工具栏  | 56px      |
| 左侧栏    | 260px，可折叠 |
| 右侧栏    | 320px，可折叠 |
| 中央阅读区  | 自适应       |
| 页面背景   | `#F3F4F6` |
| PDF 页面 | 白色，带轻微阴影  |

```txt

├──────────────────────────────┤
│ Canvas Layer                  │ PDF 页面渲染
└──────────────────────────────┘
```

# 八、核心交互流程

## 1. 上传并阅读 PDF

```txt
进入文件库
  ↓
点击上传 PDF
  ↓
选择或拖拽文件
  ↓
上传成功
  ↓
文件出现在文件库
  ↓
点击文件卡片
  ↓
进入 PDF 阅读器
```

## 2. 添加高亮

```txt
打开 PDF
  ↓
鼠标选中文字
  ↓
浮动工具栏出现
  ↓
点击“高亮”
  ↓
页面立即显示高亮
  ↓
右侧标注面板新增记录
  ↓
后台同步
```

## 3. 添加文字批注

```txt
选中文本或点击页面位置
  ↓
点击“批注”
  ↓
右侧打开批注输入框
  ↓
输入内容
  ↓
点击保存
  ↓
页面显示批注图标
  ↓
同步到云端
```

## 4. 多端继续阅读

```txt
设备 A 阅读到第 18 页
  ↓
系统保存阅读进度
  ↓
设备 B 打开同一文件
  ↓
提示：是否跳转到上次阅读位置？
  ↓
点击确认
  ↓
跳转到第 18 页
```

# 十、PC Web 端响应规则

虽然是 PC Web，但仍要考虑不同桌面宽度。

| 屏幕宽度            | 布局策略                    |
| --------------- | ----------------------- |
| ≥1440px         | 左栏、右栏默认展开               |
| 1200px - 1439px | 左栏展开，右侧标注栏可折叠           |
| 1024px - 1199px | 左栏默认折叠，仅保留图标            |
| <1024px         | 不建议作为 PC 主要体验，可提示使用更大屏幕 |

阅读器在小屏下：

```txt
左侧缩略图栏默认收起
右侧标注栏默认收起
顶部工具栏保留核心操作
标注面板通过抽屉打开
```

# 十三、最终建议

PC Web 端优先把产品设计成一个 **高效的 PDF 工作台**：

```txt
左侧：文件与页面导航
中间：PDF 阅读主区域
右侧：标注与笔记
顶部：搜索、缩放、同步、工具
```

这套结构最适合你的项目，因为它同时满足：

```txt
阅读效率
标注效率
文件管理
同步状态可见
后续功能扩展
```

推荐 MVP 页面组合：

```txt
登录页
文件库首页
PDF 阅读器页
设置页
上传弹窗
标注编辑弹窗
删除确认弹窗
```

最终产品主界面可以概括为：

> **一个面向 PC Web 的 PDF 阅读工作台，用户在左侧管理文档和页面，中间阅读 PDF，右侧管理标注与笔记，所有变更自动同步。**
